Convierte NotebookLM en un cerebro local

Exporta las fuentes de tu cuaderno, organízalas en tu computador y conviértelas en una base de conocimiento que puedas usar con agentes de IA.

En esta guía vas a aprender cómo sacar las fuentes de un cuaderno de NotebookLM, guardarlas en tu computador como archivos Markdown, organizarlas como un cerebro digital en Obsidian y usarlas con Claude Code como contexto para trabajar más rápido.

1

Descarga la extensión para exportar fuentes de NotebookLM

Para sacar las fuentes de un cuaderno de NotebookLM vamos a usar una extensión de Chrome llamada NotebookLM Sources Export.

Esta extensión sirve para exportar las fuentes de un cuaderno en formato Markdown, también conocido como archivos .md.

Importante: esta extensión funciona en Google Chrome porque es una extensión de Chrome.
Descargar NotebookLM Sources Export
Cómo fijar la extensión

Después de instalarla, fíjala en la barra superior de Chrome. Para hacerlo:

  1. Ve a la parte superior derecha del navegador.
  2. Haz clic en el ícono de pieza de rompecabezas, donde aparecen tus extensiones.
  3. Busca NotebookLM Sources Export.
  4. Haz clic en el ícono de pin para dejarla visible en la barra.
Esto es importante porque después vas a usar ese botón mientras estás dentro de tu cuaderno de NotebookLM.
2

Exporta las fuentes de NotebookLM a tu computador

Cuando quieras exportar las fuentes de un cuaderno, primero entra a NotebookLM y abre el cuaderno que quieres convertir en cerebro local.

Estando dentro del cuaderno, haz clic en la extensión NotebookLM Sources Export.

Pasos
  1. Abre tu cuaderno en NotebookLM.
  2. Haz clic en la extensión que fijaste en Chrome.
  3. Presiona el botón de extraer o exportar.
  4. Cuando termine el proceso, entra a ver la exportación.
  5. Descarga el archivo generado.

El archivo ZIP

La extensión descarga un archivo .zip.

Cuando lo descomprimes completo en tu computador, se crea una carpeta con las fuentes del cuaderno convertidas en archivos Markdown, es decir, archivos .md.

¿Qué es Markdown?

Markdown es un formato de texto limpio, liviano y fácil de leer.

Es útil porque lo entienden muy bien herramientas como Obsidian, Claude Code, Codex y otros agentes de IA.

Desde este momento, las fuentes de tu cuaderno ya no viven solo dentro de NotebookLM. Ahora también viven en una carpeta dentro de tu computador.

3

Crea la carpeta donde va a vivir tu cerebro digital

Ahora crea una carpeta nueva en tu computador para guardar todo el proyecto.

Por ejemplo, en el video usamos un proyecto sobre longevidad y buenos hábitos, así que la carpeta puede llamarse:

Proyecto longevidad
Pasos
  1. Crea una carpeta nueva en tu computador.
  2. Ponle un nombre claro según el tema del proyecto.
  3. Copia dentro de esa carpeta la carpeta que descargaste desde NotebookLM.
  4. Renómbrala como: cerebro

Resultado esperado

La estructura debe verse así:

Proyecto longevidad/ └── cerebro/ ├── fuente-1.md ├── fuente-2.md ├── fuente-3.md └── ...

La carpeta principal representa el proyecto completo.

La carpeta cerebro contiene el conocimiento exportado desde NotebookLM.

4

Organiza las fuentes como un cerebro digital

Ahora vamos a pedirle a Claude que organice esos archivos Markdown para que funcionen mejor dentro de Obsidian.

La idea no es borrar las fuentes originales, sino crear una estructura clara, conectada y navegable.

¿No tienes Claude Desktop?
  1. Entra a tu cuenta de Claude.
  2. Abre tu perfil.
  3. Ve a Apps y extensiones.
  4. Busca Desktop o Escritorio.
  5. Descarga la aplicación.

Abrir el proyecto

Luego entra a la sección de Claude Code o selecciona proyecto y elige la carpeta principal. En este ejemplo debes seleccionar:

Proyecto longevidad

Prompt principal para organizar el cerebro

Prompt para organizar el cerebro
Actúa como un arquitecto de conocimiento especializado en Obsidian. Tengo una carpeta con fuentes exportadas desde NotebookLM en formato Markdown. Quiero que conviertas esta carpeta en un cerebro digital organizado para usarlo en Obsidian y después con agentes como Claude Code o Codex. Objetivo: Organiza todo el contenido sin eliminar información importante, creando una estructura clara, navegable y conectada mediante enlaces internos de Obsidian. Instrucciones: 1. Revisa todos los archivos Markdown de la carpeta. 2. No borres el contenido original de las fuentes. 3. Crea una estructura de carpetas organizada, por ejemplo: - 00_CEREBRO - 01_FUENTES - 02_TEMAS - 03_IDEAS - 04_PROMPTS - 05_RESUMENES 4. Mueve o clasifica los archivos según corresponda. 5. Crea un archivo principal llamado `00_CEREBRO.md` que funcione como índice maestro. 6. Dentro de `00_CEREBRO.md`, resume qué contiene este cerebro digital y enlaza los temas principales usando enlaces internos de Obsidian con este formato: `[[Nombre del archivo]]`. 7. Crea notas temáticas que agrupen ideas repetidas entre varias fuentes. 8. Agrega enlaces internos entre archivos relacionados para que Obsidian pueda mostrar un mapa de nodos útil. 9. Agrega tags simples cuando tenga sentido, por ejemplo: - #ia - #contenido - #automatizacion - #investigacion - #herramientas 10. Crea un archivo llamado `PREGUNTAS_UTILES.md` con preguntas que puedo hacerle después a Claude Code o Codex usando este cerebro como contexto. 11. Crea un archivo llamado `IDEAS_DE_CONTENIDO.md` con posibles ideas de videos, posts o investigaciones basadas únicamente en las fuentes. 12. Al final, deja la carpeta lista para abrirse en Obsidian como vault. Condiciones: - No inventes información que no esté en las fuentes. - No elimines las fuentes originales. - Prioriza claridad, orden y conexión entre notas. - Usa enlaces internos de Obsidian para que se generen nodos en el graph view.

Claude Code va a revisar la carpeta, organizar las fuentes, crear índices, conectar notas relacionadas y dejar el proyecto listo para abrirlo en Obsidian.

5

Abre la carpeta en Obsidian

Descargar Obsidian

¿Qué es Obsidian?

Obsidian es una aplicación para crear bóvedas de conocimiento usando archivos Markdown.

Una bóveda es básicamente una carpeta organizada donde puedes guardar notas, fuentes, ideas, resúmenes y conexiones entre documentos.

¿Por qué usar bóvedas?

Organizar un proyecto como bóveda te permite trabajar con tus fuentes de forma más clara.

En vez de tener documentos sueltos, puedes tener un sistema conectado donde cada nota se relaciona con otras.

Pasos para abrir la bóveda
  1. Abre Obsidian.
  2. Ve a Archivo.
  3. Haz clic en Abrir bóveda.
  4. Elige la opción para abrir una carpeta como bóveda.
  5. Selecciona la carpeta principal del proyecto.

Ejemplo: Proyecto longevidad

¿Qué debería pasar?

Obsidian abrirá la carpeta como una bóveda.

Ahora vas a poder ver la estructura del cerebro digital, los archivos organizados, los enlaces internos y las conexiones entre fuentes.

Ver el mapa de nodos

Para ver el mapa del cerebro
  1. Haz clic en el botón de abrir vista gráfica.
  2. Obsidian mostrará un mapa con los nodos del proyecto.
  3. Cada nodo representa una nota o fuente.
  4. Las líneas muestran cómo se conecta la información.

Este mapa no es solo visual. Te ayuda a entender cómo se relacionan las fuentes, temas e ideas dentro de tu proyecto.

6

Usa un agente con ese cerebro

Ahora que el cerebro está organizado, puedes volver a Claude Code y abrir la carpeta del proyecto.

Al seleccionar esa carpeta, Claude puede usar los archivos Markdown como contexto para ayudarte a trabajar con ese conocimiento.

Cómo usar @

En Claude Code puedes mencionar archivos o carpetas usando @.

Esto sirve para indicarle a Claude qué parte del proyecto debe usar como referencia.

Usa @00_CEREBRO.md como fuente principal para responder.

Prompt recomendado para activar el agente

Prompt para activar el agente
A partir de ahora, usa `@00_CEREBRO.md` como índice principal de conocimiento de este proyecto. Tu tarea es actuar como un agente de investigación y apoyo basado en este cerebro digital. Reglas: 1. Usa el contenido del cerebro como fuente principal para responder. 2. Cuando necesites contexto, revisa primero `@00_CEREBRO.md` y luego los archivos relacionados que estén enlazados desde ahí. 3. No inventes información que no esté en las fuentes. Si algo no está en el cerebro, dilo claramente. 4. Si durante nuestra conversación aparece información nueva, una decisión importante, una idea útil o una corrección relevante, guárdala en archivos `.md` dentro de la carpeta adecuada del proyecto. 5. Mantén el cerebro organizado. Usa nombres de archivo claros, carpetas coherentes, enlaces internos de Obsidian y tags cuando tenga sentido. 6. Si una nueva información conecta con una nota existente, actualiza esa nota o crea un enlace interno hacia ella. 7. Antes de crear archivos nuevos, revisa si ya existe una nota donde esa información pueda encajar mejor. 8. Cuando respondas, prioriza claridad, utilidad y conexión con las fuentes del proyecto. 9. Al final de respuestas importantes, sugiere si conviene guardar algo nuevo en el cerebro. 10. Cada vez que guardes información nueva, indícame brevemente qué archivo actualizaste o creaste. Objetivo: Que este proyecto funcione como un cerebro vivo, que no solo responde usando las fuentes actuales, sino que también aprende de las interacciones importantes y mantiene organizada la información nueva.

Con esta instrucción, Claude no solo usa el cerebro como referencia. También puede ayudarte a mantenerlo actualizado, creando o modificando archivos Markdown cuando aparezca información importante.
